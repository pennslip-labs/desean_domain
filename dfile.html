dFile

Retro-Styled File Manager

A lightweight, efficient file manager with a nostalgic console-like theme. Built with C++ and wxWidgets for cross-platform performance.

About This Project

dFile is a file manager application that combines retro aesthetics with practical file management features. Inspired by vintage terminal interfaces and early GUI file managers, it provides an intuitive way to navigate, organize, and manage files on your system. With its minimal resource footprint and nostalgic design, dFile brings back the charm of early computing while remaining fully functional and modern.

Core Features

Retro Console Aesthetic

Classic terminal-inspired UI with modern responsiveness. Experience computing as it was meant to be.

Efficient Navigation

Browse directories with intuitive sidebar navigation and organized file list view.

File Details

View comprehensive information about selected files including size, date modified, and permissions.

Search & Filter

Quickly find files with powerful search and filter functionality.

Lightweight

Minimal resource footprint with blazing fast performance on any system.

Cross-Platform

Runs seamlessly on Windows, macOS, and Linux with native performance.

Technical Details

Technology Stack

  • Language: C++ (C++11 standard)
  • GUI Framework: wxWidgets 3.2+
  • Platforms: Windows, macOS, Linux
  • License: GNU General Public License v3.0

Key Components

  • Launcher: wxApp initialization & CLI arguments
  • Frame: Main window and UI layout
  • Controller: Business logic & event handling
  • DirectoryScanner: File system traversal
  • Panels: Sidebar, file list, details, filter

System Requirements

  • C++11 compatible compiler
  • 512MB RAM minimum
  • 50MB disk space for build
  • wxWidgets 3.2 or later

Build & Installation

Dependencies

Linux (Ubuntu/Debian):

sudo apt-get install libwxgtk3.2-dev build-essential

macOS:

brew install wxwidgets

Windows: Download MSVC Build Tools and wxWidgets from wxwidgets.org

Compiling from Source

  1. Clone the repository: git clone https://github.com/slipnut44/dFile.git
  2. Navigate to the cpp_files directory: cd dFile/cpp_files
  3. Compile using Make: make
  4. Run the executable: ./bin/dFile

Using CMake (Optional)

  1. mkdir build && cd build
  2. cmake ..
  3. make
  4. ./dFile

Command Line Usage

dFile [DIRECTORY]

Example: dFile ~/Documents
